Project Overview: Pastry Portfolio for Chef Jazmin Lester

Application Description

This project is a professional pastry portfolio website designed for Pastry Chef Jazmin Lester. The site showcases her custom cakes, desserts, and bakery collaborations while also presenting her flavor offerings, menu options, and weekly schedule. The primary purpose of the application is to increase her online visibility and provide potential customers with a polished and organized way to explore her work and submit non-transactional order inquiries.

The intended users include local customers searching for custom desserts, event planners organizing special occasions, bakery owners seeking collaboration opportunities, and returning customers or fans browsing new work. The website content includes a biography, a dessert gallery, a menu of flavors and pricing guidance, a structured inquiry form, her weekly schedule, location information, and an optional testimonials or FAQ section.

Client Information

The client for this project is Pastry Chef Jazmin Lester, the owner of Jazzed Up Desserts, LLC. She can be contacted at jazzedupdessertsllc@gmail.com, and her phone number has been marked as private for confidentiality. Chef Jazmin is an independent pastry chef specializing in custom cakes and luxury dessert creations. Her primary goal for the website is to have a visually appealing and intuitive platform that highlights her baking expertise and makes it easier for customers to understand her offerings and request custom orders.

Page Design Details

Each page in the pastry portfolio website follows a consistent structure that includes the shared header, navigation bar, footer, and responsive content areas. The Home page introduces the brand and welcomes the user with a hero banner, a brief introduction to Chef Jazmin, and images of featured desserts that open in a modal viewer. The main purpose of this page is to attract and orient visitors, and it does not include any data-entry fields or validation. Animation or soft transitions may be used to enhance the user experience.

The About page presents Chef Jazmin’s background, experience, and baking philosophy. It is designed for users who want to learn more about the chef before placing an inquiry. The page includes narrative content, portrait images, and a mission statement. It does not include form fields or validation. Internal navigation links guide users toward exploring the menu or submitting an order inquiry, and optional visual effects can accompany major content sections.

The Menu and Gallery page serves customers who wish to browse dessert categories or view images of previous creations. It includes interactive dessert categories organized with jQuery UI tabs and high-quality images that open in a lightbox-style viewer. No data-entry fields appear on this page, and no validation is required. Visitors can switch among categories, explore the gallery, and access ordering options from within the page.

The Order Inquiry page supports customers who want to request custom dessert orders. It includes a structured form with fields for the customer’s name, email, dessert type, flavor selection, event date, serving size, and additional notes. JavaScript provides client-side validation for required fields, email formatting, and valid date selection, and the date picker is implemented using jQuery UI. A confirmation message appears upon successful submission, though no data is stored to comply with the course’s no-CRUD requirement.

The Schedule and Location page offers users a clear view of Chef Jazmin’s weekly operating hours along with a map showing her bakery pickup or consultation location. The schedule is loaded dynamically from a JSON file using AJAX, which allows content updates without modifying the HTML markup. The page does not use input fields or validation and includes an embedded Google Map for geographic context.

The Testimonials and FAQ page strengthens trust and addresses the common questions customers may have about orders, timelines, pricing, and quality. This page contains customer feedback and an accordion-style FAQ powered by jQuery UI, allowing users to expand and collapse each question. No forms or validation appear on this page, and its interactive components focus solely on improving user comprehension.

Planned Dynamic Functionality

The website incorporates multiple layers of interactivity using JavaScript, jQuery, jQuery UI, and AJAX. One of the core dynamic components is the interactive photo gallery, where dessert images open in a lightbox-style modal viewer to allow visitors to explore each item in detail. A similar concept can be seen in the lightbox example provided by W3Schools at this link.

The Menu and Gallery page includes jQuery UI tabs to organize dessert categories into clear groupings. This helps users explore multiple sections without scrolling through an overwhelming amount of content. The jQuery UI demonstration at this link shows the inspiration for this feature.

The FAQ section on the Testimonials page uses a jQuery UI accordion to provide an expandable question-and-answer interface. This format improves readability and reduces visual clutter by allowing users to expand only the questions that interest them. A reference implementation can be seen at this demo.

The Order Inquiry page uses JavaScript to validate fields such as name, email format, and date selection. The purpose of this functionality is to improve data quality before the user submits the form. This approach mirrors the validation techniques illustrated at this reference example.

The Schedule and Location page incorporates an AJAX request to load weekly hours stored in a JSON file. This method allows the weekly schedule to remain easily editable without making structural changes to the HTML. A similar AJAX pattern is demonstrated at this link.
