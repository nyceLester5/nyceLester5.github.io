Project Overview: Pastry Portfolio for Chef Jazmin Lester

Application Description

This project is a professional portfolio website for Pastry Chef Jazmin Lester, focused on showcasing her custom cakes, desserts, and bakery collaborations. The application centralizes her visual work, menu offerings, and weekly availability, while providing a structured way for customers to submit non-transactional order inquiries.

The website’s purpose is to increase Jazmin’s online visibility, present her work in a clean and modern layout, and streamline how potential clients learn about her services, flavor options, and schedule.

Intended Users

  • Local customers seeking custom cakes and dessert orders for events.
  • Event planners organizing weddings, birthdays, showers, and corporate events.
  • Bakery owners or managers interested in collaborations.
  • Friends, followers, and returning customers browsing her portfolio.

Overview of Website Content

  • Chef biography and brand story.
  • Photo gallery of cakes, cupcakes, and specialty desserts.
  • Menu with flavor options and starting prices.
  • Order inquiry form (front-end only, no data storage).
  • Weekly schedule and bakery location map.
  • Optional testimonials and frequently asked questions.

Page Design Details

Each page will use a consistent layout that includes the shared header, navigation bar, footer, and responsive page structure. The following sections describe the design requirements for each individual page.

1. Home (index.html)

  • Purpose: Introduce the brand and highlight featured desserts to attract users immediately.
  • Audience/Users: All site visitors including new customers, returning customers, and event planners.
  • Content: Hero banner, tagline, chef introduction, featured dessert images.
  • Data Fields: None.
  • Validations: Not applicable.
  • Buttons/Links: “Browse Gallery”, “Order Inquiry”, main navigation links.
  • Actions: Clicking featured images opens a lightbox modal; buttons link to additional pages.
  • Special Notes: Page may include subtle animations to improve first impressions.

2. About (about.html)

  • Purpose: Share Chef Jazmin’s background, qualifications, and baking philosophy.
  • Audience/Users: Clients who want to verify credibility or learn her story.
  • Content: Biography text, portrait photo, mission statement, brand story.
  • Data Fields: None.
  • Validations: Not applicable.
  • Buttons/Links: Links to Menu & Gallery and Order Inquiry pages.
  • Actions: Smooth scrolling and fade-in effects for text sections (optional).
  • Special Notes: Focus on storytelling and building trust.

3. Menu & Gallery (menu-gallery.html)

  • Purpose: Display available dessert categories and showcase high-quality images of past work.
  • Audience/Users: Customers comparing flavors, styles, and dessert offerings.
  • Content: Menu flavor list, tabs for dessert categories, images with descriptions.
  • Data Fields: None.
  • Validations: Not applicable.
  • Buttons/Links: Category tabs, filter options, “Request This Style” button leading to Order Inquiry.
  • Actions: jQuery UI tabs allow switching between categories; images open in a modal viewer; filters adjust visible items.
  • Special Notes: This page uses the most interactive visual elements.

4. Order Inquiry (order.html)

  • Purpose: Collect front-end only custom order details from visitors.
  • Audience/Users: Customers interested in placing custom dessert requests.
  • Content: Inquiry form containing name, email, dessert type, flavor, event date, serving size, and notes.
  • Data Fields: Multiple input fields, dropdowns, textarea, date picker.
  • Validations: Required field checks; email format validation; date validation using JavaScript.
  • Buttons/Links: “Submit Inquiry”, “Clear Form”, navigation links.
  • Actions: JavaScript displays success message or errors; jQuery UI datepicker opens; form resets after submission.
  • Special Notes: No user data is stored due to the course’s no-CRUD rule.

5. Schedule & Location (schedule.html)

  • Purpose: Display weekly bakery hours and provide a visual map for pickups or consultations.
  • Audience/Users: Customers planning visits or coordinating order pickup.
  • Content: Weekly schedule loaded from JSON, Google Map embed, address details.
  • Data Fields: None.
  • Validations: Not applicable.
  • Buttons/Links: Link to external Google Maps for directions.
  • Actions: AJAX request retrieves JSON schedule; map allows zoom and pan.
  • Special Notes: Keeps operating hours flexible to update without editing HTML.

6. Testimonials / FAQ (testimonials.html)

  • Purpose: Provide social proof and address common customer questions.
  • Audience/Users: New customers and event planners evaluating service quality.
  • Content: Customer quotes, frequently asked questions, accordion-style toggles.
  • Data Fields: None.
  • Validations: Not applicable.
  • Buttons/Links: Collapsible accordion headers, links to other site pages.
  • Actions: jQuery UI accordion opens/closes selected panels.
  • Special Notes: Optional page but recommended for brand credibility.

Planned Dynamic Functionality

The website will use JavaScript, jQuery, jQuery UI, and AJAX to support interactive and user-friendly features. The following list describes each planned dynamic component, its purpose, and an example of a website demonstrating similar functionality.

  • Interactive Photo Gallery (Lightbox Viewer)
    Description: Dessert images open in a modal overlay when clicked.
    Purpose: Enables users to view cake designs in detail without leaving the page.
    Example: W3Schools Lightbox Example
  • jQuery UI Tabs for Menu Categories
    Description: Organizes dessert categories (cakes, cupcakes, treats) into tabbed panels.
    Purpose: Reduces page clutter and improves menu navigation.
    Example: jQuery UI Tabs Demo
  • jQuery UI Accordion for FAQs
    Description: FAQ items expand or collapse when clicked.
    Purpose: Allows users to quickly find answers without navigating multiple pages.
    Example: jQuery UI Accordion Demo
  • Validated Order Inquiry Form
    Description: JavaScript ensures required fields are complete and properly formatted.
    Purpose: Improves data quality and guides the customer through the process.
    Example: JavaScript Form Validation Example
  • AJAX-Loaded Weekly Schedule
    Description: A JSON file containing weekly operating hours is loaded dynamically using AJAX.
    Purpose: Allows schedule updates without editing HTML pages and improves maintainability.
    Example: AJAX getJSON Example
